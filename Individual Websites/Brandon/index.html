Project

CUDA-SQUEEZE is a UCT Computer Science Honours comression project for SKA (Square Kilometer Array) South Africa.

Aim

The aim is to create an algorithm that can compress the data from the SKA dishes for storage in "Real Time" (Without the compression slowing it down). The current speed of data input from the dishes is 2 terra-samples of 10-bit floating point data per second, sent through FPGAs to convert each value to a 32-bit floating point value (Figure 1). This is equivalent to 2.2 Terra-bytes (TB) per second. Since the data is then send through networks, this is brought down to an aproximate 36 Giga-Bytes (GB) per second. Thus in order to acheive the "Real Time" compression we will need to acheive an aproximation of 37GB per second of incoming data.

Process

In order to find the best possible algorithm for SKA the Honours Group I am in decided to make 3 different algorithms (One for each person in the group):

  • Benjamin Hugo -> Predictive method
  • Brandon Talbot -> Huffman Coding
  • Heinrich Strauss -> Zero Length Coding

A sequential CPU algorithm, fully multithreaded CPU algorithm and a GPU algorithm will be made for all compression methods listed above. This will allow us to determine if the algorithm is worth running on the GPU, or even if the CPU method is a good candidate.
It will also allow us to compare algorithms to determine the best possible method for SKA's needs.
